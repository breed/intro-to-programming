# Why?

![another mid c++ book](images/c++books++.png)

There are lots of programming text books.
Why in the world would we do another?!?!?!
Tragically they are either mediocre or super advanced, and most are out of date since languages are evolving so quickly.
I once heard Linux Torvalds say ~1996 at USCS: I believe there will always be a niche for a free version of UNIX.
(That is what I remember at least...)
I believe there will always be a niche for free programming textbooks that evolve with the languages.

We want the books we use the match our curricula needs, so having control over our books helps.

![meh c++ book](images/mehc++book.png)

Perhaps, the book is *meh*.
We are going for engaging, but meh can work as long as it conveys what we need.

# Books for introductory C/C++ programming classes

- [Gorgo Starting C++](sc++) introduces the non-programmer to C++
- [Gorgo C for C++ Programmers](c4c++) introduces C++ programmers to C. Good for an intro C/C++ class where students need both languages. Or for an Operating Systems class where students know C++  (or Java) but not C.
- Mastering C++ (TBD) object-oriented programming with C++ and advanced built-in data structures.

# Contributions

Do you see something missing? An API that needs covering? A better analogy? Please let Gorgo know. She loves feedback! (Most wise people do!)

Open an issue to give feedback. This is much better than doing a PR.
**Gorgo has a voice. Everything flows better if we keep things in Gorgo's voice.
Even change we make directly get rewritten by Gorgo.

If your issue does get implemented in the text, we will make sure that Gorgo marks you as an author or co-author so that you show up as a contributor on the project and the activity is registered on your github activity.

**Do not use other people's work in an issue! We don't want to plagiarize works of others. If we want to quote something, we need to make the quote and source clear as in the intro for c4c++.**
